Basada en la consulta anterior calcular el monto monetario de la comisión en una nueva columna.

- Si tiene comisión → `salary × commission_pct` (redondeado a 2 decimales)
- Si es `NULL` → devuelve `0`

```
SELECT
    first_name,
    last_name, 
    salary, 
    commission_pct, 
    CASE 
        WHEN commission_pct IS NULL  THEN 'Sin comisión'
        WHEN commission_pct <  0.15 THEN  'Comisión baja' 
        WHEN commission_pct <  0.30 THEN  'Comisión media' 
        ELSE                              'Comisión alta'
    END AS nivel_comisión, 
    CASE 
        WHEN commission_pct IS NOT NULL THEN
        ROUND(salary * commission_pct, 2) 
        ELSE 0 
    END AS “importe comisión" 
FROM   employees 
ORDER BY commission_pct;
```

En base a la consulta anterior agregar una nueva columna que muestra el porcentaje de comisión como texto legible para llevarlo a un reporte.:

- `NULL` → `'0%'`
- Si tiene valor → multiplica por 100 y concatena `%` (ej:  `elvalor 0.15` → `es llevado a 15%`)

```
SELECT first_name, last_name, salary,
CASE 
    WHEN commission_pct IS NULL THEN '0%'
    ELSE to_char(commission_pct*100) || '%'
END AS " % Comisión",
CASE
    WHEN commission_pct IS NULL THEN 'SIN COMISION'
    WHEN commission_pct < 0.15 THEN 'BAJA COMISION'
    WHEN commission_pct >= 0.15 AND commission_pct <= 0.3 THEN 'COMISION MEDIA'
    ELSE 'COMISION ALTA'
END AS "Tipo Commission",
CASE
    WHEN commission_pct IS NULL THEN 0	
    ELSE round(commission_pct * salary,2)
END AS "Valor comisión (en dolares)"
FROM employees;
```

Quiero ver el salario máximo de cada departamento junto al legajo del empleado que cobra dicho salario:

**PRIMER INTENTO (ERROR!!)**

```
SELECT employee_id, department_id,
       salary,
       MAX(salary)  
FROM   employees
GROUP BY department_id;  -- ERROR: employee_id no está en GROUP BY
```

En este primer intento vemos que el employee_id no esta en el GROUP BY. Oracle  lanzará el famoso error: ORA-00979: not a GROUP BY expression.

**SEGUNDO INTENTO (SE APROXIMA BASTANTE)**

```
SELECT 
    employee_id,
    department_id,
    salary AS salario_actual,
    MAX(salary) OVER(PARTITION BY department_id) AS salario_max_depto
FROM 
    employees;
```

En este segundo intento logramos que aparezca tantas filas como empleados dentro de la organización y logramos que en una nueva columna aparezca el salario máximo por departamento. 

**TERCER INTENTO (ESTAMOS CERCA DE LA SOLUCIÓN)**

```
SELECT 
    employee_id,
    department_id,
    salary AS salario_actual,
    MAX(salary) OVER(PARTITION BY department_id) AS salario_max_depto,
    RANK() OVER (PARTITION BY department_id ORDER BY salary desc) AS "Puesto"
FROM 
    employees
WHERE 
     department_id IS NOT NULL;

```

RANK() agrega una nueva columna donde asigna un número de posición a cada empleado dentro de su departamento, ordenado de mayor a menor salario.
