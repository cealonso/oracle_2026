La empresa necesita un reporte de sus empleados. Crear una vista llamada v_emp_jh que muestre, por cada empleado, su nombre completo, fecha de ingreso, salario y el departamento al que pertenece. Además, si el empleado ocupó puestos anteriores en la empresa, mostrar el código del último puesto que tuvo y la fecha en que lo dejó. 

```
CREATE or REPLACE VIEW v_emp_jh AS
SELECT
    e.employee_id,
    e.first_name || ' ' || e.last_name   AS full_name,
    e.hire_date,
    e.salary,
    d.department_name,
    jh.job_id       AS prev_job_id,
    jh.end_date     AS prev_end_date
FROM       employees   e
-- (solo aparecen empleados que tienen departamento asignado)
INNER JOIN departments d  ON e.department_id = d.department_id
-- (aparecen los empleados que alguna vez rotaron en la empresa)
INNER JOIN  job_history jh ON e.employee_id   = jh.employee_id
AND
jh.end_date=(select max(end_date) from job_history where e.employee_id=employee_id)
```

Usando la vista anterior mostrar el nombre completo, y el salario de los empleados del departamento Sales:

```
SELECT full_name, salary FROM v_emp_jh WHERE department_name = 'Sales';
```

Crear una vista denominada v_info_emp que muestra un reporte de empleados combinando datos de tres tablas y agregando dos columnas calculadas.  Las columnas a devolver son las siguientes:

- Nombre completo concatenado  first_name` y `last_name.
- Nombre del departamento y título del puesto.
- Fecha de ingreso y salario redondeado a 2 decimales.
- salary_rank para  rankear a cada empleado por salario (de mayor a menor) dentro de su propio departamento.
- salary_band clasifica el salario en cuatro categorías :

|Rango|Banda|
|---|---|
|Menos de $5.000|Junior|
|$5.000 – $9.999|Mid|
|$10.000 – $14.999|Senior|
|$15.000 o más|Executive|

```
CREATE or REPLACE VIEW v_employees AS
SELECT
    e.employee_id,
    e.first_name || ' ' || e.last_name  AS full_name,
    d.department_name,
    j.job_title,
    e.hire_date,
    round(e.salary,2) as salary,
    RANK() OVER (
        PARTITION BY e.department_id
        ORDER BY e.salary DESC) AS salary_rank,
    CASE
        WHEN e.salary < 5000  THEN 'Junior'
        WHEN e.salary < 10000 THEN 'Mid'
        WHEN e.salary < 15000 THEN 'Senior'
        ELSE 'Executive'
    END AS salary_band
FROM employees e
JOIN departments d ON e.department_id = d.department_id
JOIN jobs j ON e.job_id=j.job_id;

```
