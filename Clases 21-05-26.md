
Crear la tabla employees_courses para almacenar los empleados que realizaran capacitaciones:

```
CREATE TABLE employees_courses (
    training_id NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY 
    PRIMARY KEY,    
    employee_id NUMBER NOT NULL,
    course_id NUMBER NOT NULL, 
    is_finished BOOLEAN NOT NULL, 
    CONSTRAINT fk_employee_id FOREIGN KEY (employee_id)
    REFERENCES employees (employee_id),
    CONSTRAINT fk_course_id FOREIGN KEY (course_id)
    REFERENCES courses (course_id) 
   );
```

Usar la instrucción CASE de SQL para  realizar lo siguiente:

- Si el salario es menor a 6000 se creará una nueva columna con el texto Salario Bajo. 
- Si el valor se encuentra entre 6000 y es menor de 10000 el texto que aparecerá será Salario Medio. 
- Si el salario es mayor a 10000 el texto que finalmente que aparecerá será Salario Alto

```
SELECT first_name, last_name, salary,
CASE
WHEN salary<6000 THEN 'Bajo'
WHEN salary>=6000 AND salary<10000 THEN 'Medio'
WHEN salary>=10000 THEN 'Alto'
END AS Tipo
FROM employees
ORDER BY first_name ASC;
```

Agregar un ELSE para simular la instrucción anterior:

```
SELECT first_name, last_name, salary,
CASE
WHEN salary<6000 THEN 'Bajo'
WHEN salary>=6000 AND salary<10000 THEN 'Medio'
ELSE 'Alto'
END AS Tipo
FROM employees
ORDER BY first_name ASC;
```

Si el porcentaje es menor al 15% devolver el valor BAJA COMISION.
Si el porcentaje se encentra entre el 15% y el 30% devolver el valor  COMISION MEDIA.
Si el porcentaje supera el 30% colocar el mensaje COMISION ALTA.

Además se pide colocar un rotulo a la columna de acuerdo a la información que se este mostrando:

```
SELECT first_name, last_name, salary, commission_pct,
CASE
WHEN commission_pct IS NULL THEN 'SIN COMISION'
WHEN commission_pct<0.15 THEN 'BAJA COMISION'
WHEN commission_pct>=0.15 AND commission_pct<=0.3 THEN 'COMISION MEDIA'
ELSE 'COMISION ALTA'
END AS "Tipo Commission"
```
